# 1.8: iOS Test Harness — `make test-ios` chạy test thật (bridge + build-smoke)

## Story

As a kỹ sư iOS trên `mindful-key`,
I want `make test-ios` chạy được một bộ chứng cứ thật gồm (a) test tầng bridge tái dùng đúng
5 ca Telex→Unicode đã có trong `tests/core/test_engine.cpp` và (b) một bước build-smoke
`xcodebuild ... MindfulKeyKeyboard -sdk iphonesimulator`, gate bằng exit code như `test-core`,
So that đội iOS có lưới an toàn thật chứng minh cầu nối `KeyboardBridge` + target extension vẫn
build/gõ đúng sau mỗi thay đổi, thay vì dựa vào README nói suông "no-op".

1. `make test-ios` chạy `tests/ios/bridge_test` (test tầng bridge) và bài test này gõ ĐÚNG 5 ca
   Telex→Unicode đã có sẵn trong `tests/core/test_engine.cpp` (`"xin chaof cacs banj"→"xin chào
   các bạn"`, `"tieengs vieetj"→"tiếng việt"`, `"tooi ddang vui"→"tôi đang vui"`, `"tooi ddang
   buoonf"→"tôi đang buồn"`, `"hoom nay meejt quas"→"hôm nay mệt quá"`) đi XUYÊN QUA
   `KeyboardBridge` (không gọi thẳng `vKeyHandleEvent`) — không được bịa thêm ca mới ngoài 5 ca
   này cho story này. Cả 5 ca PASS, exit code 0.
2. `make test-ios` CÒN chạy một bước build-smoke: `xcodegen generate` (sinh lại
   `.xcodeproj` từ `project.yml`) rồi `xcodebuild -project platforms/apple/MindfulKey.xcodeproj
   -scheme MindfulKeyKeyboard -sdk iphonesimulator build` — hoàn tất với 0 error.
3. Target `test-ios` trong `Makefile` không còn là no-op/echo — nó exit code khác 0 nếu MỘT
   TRONG HAI bước (bridge test HOẶC build-smoke) thất bại, đúng kiểu gate exit-code mà
   `test-core` đang dùng (để `make test` — chạy `test-core test-macos test-ios` — dừng đỏ khi
   iOS regression).
4. `tests/ios/README.md` được cập nhật khớp trạng thái THẬT hiện tại (bỏ câu "trống — chưa có vỏ
   iOS... no-op" đã lỗi thời) — mô tả `make test-ios` chạy gì (bridge test + build-smoke) và có
   thêm mục ghi kết quả kiểm RAM thủ công qua Instruments (xem Dev Notes — không tự động hoá
   được việc "hệ điều hành có kill tiến trình không").
5. `git diff core/` rỗng và không file nào dưới `platforms/apple/macos/*.mm` bị đụng bởi story
   này.

## Testing

<!-- LOCKED. External dev tools must not edit Testing. -->

- **Baseline trước khi sửa gì:** chạy `make test-ios` hiện tại (chỉ có bridge_test), ghi lại
  output/exit code làm mốc so sánh — xác nhận đúng như Dev Notes mô tả (5 ca PASS) trước khi thêm
  build-smoke.
- **Build-smoke — unit-of-verification riêng:** chạy tay `xcodegen generate` rồi `xcodebuild
  -project platforms/apple/MindfulKey.xcodeproj -scheme MindfulKeyKeyboard -sdk iphonesimulator
  build` NGOÀI `make test-ios` trước, xác nhận 0 error, trước khi wire vào Makefile/script — tách
  bước "verify lệnh đúng" khỏi "verify wiring đúng".
- **Gate exit-code (AC #3) — fault injection có chủ đích:** đây là cách DUY NHẤT chứng minh gate
  hoạt động thật (không chỉ "trông có vẻ đúng"): tạm sửa 1 ca kỳ vọng sai trong `bridge_test.mm`
  (hoặc gây lỗi compile 1 dòng ở `KeyboardBridge.mm`) → chạy `make test-ios` → xác nhận exit code
  ≠ 0 và output in ra lỗi rõ ràng → revert nguyên trạng, KHÔNG commit bản đã phá.
- **Regression toàn cục:** chạy `make test` (chuỗi `test-core test-macos test-ios`) sau khi hoàn
  tất, xác nhận cả 3 đội vẫn xanh và không giẫm chân đội macOS (đúng NFR "không giẫm chân" ở
  tech-spec §Risk R2).
- **Không cần fixture/mock ngoài** — bridge test tự chứa (định nghĩa extern config + biên dịch
  cùng nguồn `core/engine`), build-smoke chỉ cần Xcode + `xcodegen` cài sẵn trên máy dev (đã có,
  dùng cho `make build`/`make generate` macOS).
- **Edge case cố ý KHÔNG tự động hoá** — hành vi "hệ điều hành kill tiến trình do RAM" không unit-
  test được (đúng theo tech-spec §Performance/Load); story chỉ chuẩn bị chỗ ghi log thủ công
  (README), KHÔNG viết assertion giả để "coi như đã test".
- **Không viết thêm ca Telex mới** cho bridge test trong story này — phạm vi khoá cứng ở AC #1 là
  tái dùng đúng 5 ca đã có, không mở rộng ma trận test (việc đó thuộc phạm vi story khác nếu cần).
