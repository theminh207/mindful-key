## Story

As a **platform-shell developer lắp ráp panel macOS** (người sẽ code story 1.3–1.6),
I want **3 control AppKit tái dùng theo đúng token NOW BRAND OS: `PillSwitch`, `StatusDot`, `CTAButton`**,
so that **các card sau (Gác cổng, Chuông, Bộ gõ) không phải tự implement lại tint/màu/WCAG mỗi lần, và không ai vô tình vi phạm hiến chương (đèn xanh-lá, chữ trắng trên cam, dot đa nghĩa) vì logic đã khóa sẵn trong 1 chỗ.**

---

## Acceptance Criteria

<!-- LOCKED. External dev tools must not edit Acceptance Criteria. -->

1. **`PillSwitch`** tồn tại trong `BrandControls.h/.m` là 1 control NSSwitch-based tái dùng: state **on** render tint `teal` (`Brand teal`, không phải `NSColor systemGreen`/tint mặc định hệ thống), state **off** render màu `divider` (xám `#E5E7E8`), **focus** (bàn phím) hiện outline 2px `teal` offset 2px, **disabled** render ở 40% opacity. Cả 4 state phải verify được bằng mắt (build app thật, Tab tới control, toggle on/off, disable thử).

2. **`StatusDot`** là 1 dot **đúng 1 màu duy nhất** theo trạng thái nhị phân: `on` = tô đặc `teal`, `off` = chỉ viền `divider` không tô nền. API public của `StatusDot` **CHỈ nhận 1 tham số boolean bật/tắt** (vd `setOn:(BOOL)isOn`) — không có tham số biên độ/mức độ/màu nào khác, để không component nào sau này có thể lỡ wire nó vào dữ liệu cảm xúc.

3. **`CTAButton`** render nền `cta.orange` `#FF7A1A` với **chữ TỐI** `text.primary` `#2A2A2A` ở MỌI state (default/hover/active/disabled) — tuyệt đối không có đường code nào set chữ trắng trên nền cam. Radius `radius.control` 8px.

4. `CTAButton` có đủ 4 state phân biệt được bằng mắt: **hover** (cam đậm nhẹ hơn), **focus** (outline 2px offset 2px), **active** (dịch xuống 1px lúc nhấn), **disabled** (40% opacity) — expose qua property/method để code gọi (story 1.3–1.6) có thể trigger từng state.

5. Không file nào trong `BrandControls.h/.m` dùng SF Symbol `face.*`, `exclamationmark.triangle`, `xmark.octagon`, hay bất kỳ `NSColor systemGreen/systemRed/systemYellow` nào để biểu đạt trạng thái — grep-verifiable, 0 match.

6. `BrandControls.h` là header public độc lập (import được mà không cần `ViewController.m`/storyboard), 3 class compile sạch trong `make build` (0 error, không sinh thêm warning mới so với baseline trước story), và **không sửa** `BrandColors.h`/`BrandColors.m` — chỉ đọc qua `Brand` class có sẵn.

## Dev Notes

<!-- LOCKED. External dev tools must not edit Dev Notes. -->

**Component spec nguồn:**
- `PillSwitch`: "NSSwitch với tint override = teal (KHÔNG để xanh-lá hệ thống mặc định — né tái tạo cặp đèn xanh/đỏ bị cấm). Off = xám divider. States: on/off/focus (outline 2px teal)/disabled (40% opacity)." [Source: DESIGN-macos-control-panel.md#2.5]
- `StatusDot`: "dot cạnh header CHỈ báo bộ gõ đang bật/tắt tiếng Việt — nhị phân, 1 màu (teal khi bật, viền divider no-fill khi tắt). TUYỆT ĐỐI không dùng dot này cho cảm xúc." [Source: DESIGN-macos-control-panel.md#2.4]
- `CTAButton`: "Nền cta.orange #FF7A1A + chữ TỐI text.primary #2A2A2A (KHÔNG chữ trắng — chữ trắng trên cam chỉ đạt 2.61:1, trượt AA; chữ tối đạt 5.50:1). Radius.control 8px. States: hover (cam đậm nhẹ), focus (outline 2px), active (nhấn xuống 1px), disabled (40%). Cam CHỈ xuất hiện ở đây và link active." [Source: DESIGN-macos-control-panel.md#2.6]

**Token màu dùng (đọc qua `Brand` class có sẵn, KHÔNG tự hard-code hex):**
- `teal #1D7C91`, `divider #E5E7E8`, `cta.orange #FF7A1A`, `text.primary #2A2A2A` [Source: DESIGN-macos-control-panel.md#1.1]
- Codebase thật đã có `Brand` class (`platforms/apple/macos/BrandColors.h/.m`) expose `+teal`, `+orange`, `+charcoal`, đọc từ Assets.xcassets Color Set, với comment gốc "KHÔNG hard-code hex ở đây để tránh lệch giữa asset catalog và code khi brand đổi màu" — story này PHẢI tuân thủ convention đó, gọi `[Brand teal]`/`[Brand orange]`/`[Brand charcoal]` thay vì literal hex. `divider #E5E7E8` chưa có accessor trong `Brand` hiện tại — nếu thiếu, thêm 1 named Color Set mới trong Assets.xcassets tên `Divider` và **có thể** thêm accessor `+ (NSColor *)divider` vào `BrandColors.h/.m` là ngoại lệ DUY NHẤT được phép chạm file đó, giữ tối thiểu (1 property, không đổi gì khác). [Inference — suy từ pattern các accessor hiện có, DESIGN không nói rõ có sẵn accessor divider hay chưa]

**Hợp đồng WCAG (bắt buộc, đã verify bằng số thật):**
- `white / orange #FF7A1A` = 2.61:1 → **FAIL**, cấm dùng.
- `text.primary #2A2A2A / orange #FF7A1A` = 5.50:1 → PASS AA. [Source: DESIGN-macos-control-panel.md#3]
- Focus indicator bắt buộc: outline 2px teal solid, offset 2px, trên MỌI control reachable. [Source: DESIGN-macos-control-panel.md#3]
- Target size ≥ 28×28pt + spacing ≥ 8px (chuẩn macOS pointer, không cần 44px touch). [Source: DESIGN-macos-control-panel.md#3]

**Ràng buộc HIẾN CHƯƠNG áp cho story này:**
- NSSwitch/toggle liên quan mood override tint về teal, không xanh-lá hệ thống — áp dụng cho `PillSwitch` dù ở story này nó chưa wire vào mood cụ thể (đó là việc của 1.5/1.6), nhưng control nền tảng phải sẵn đúng tint ngay từ đầu. [Source: sharding-context.md ràng buộc #8; DESIGN-macos-control-panel.md#5.5]
- Cam `#FF7A1A` CHỈ dùng cho CTA + link active, không ở đâu khác — `CTAButton` là nơi DUY NHẤT trong bộ control này được phép dùng cam. [Source: sharding-context.md ràng buộc #3; DESIGN-macos-control-panel.md#2.6, #5.6]
- Cấm SF Symbol `face.*`/`exclamationmark.triangle`/`xmark.octagon`, cấm status dot 2 màu, cấm đèn đỏ/vàng/xanh-lá mã hóa trạng thái. [Source: sharding-context.md ràng buộc #1; DESIGN-macos-control-panel.md#2.2, #5.1, #5.2]
- `StatusDot` là dot kỹ thuật nhị phân (bật/tắt gõ tiếng Việt) — KHÔNG BAO GIỜ được tái dùng cho biểu đạt cảm xúc; đó là lý do AC#2 khóa cứng chữ ký API chỉ nhận boolean. [Source: DESIGN-macos-control-panel.md#2.4]

**Phạm vi story này KHÔNG bao gồm:**
- Không animate biên độ (đó là `EmotionWave`, story 1.2, file riêng `EmotionWaveView.*`) — 3 control ở đây là control tĩnh theo state rời rạc, không phải glyph sóng. [Inference — theo phân chia Wave 1 trong sharding-context.md]
