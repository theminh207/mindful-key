Understand-Anything

Bản đồ tri thức (knowledge graph) cho code mindful-key — quét 1 lần, đọc hiểu nhanh nhiều lần. Chỉ đọc hiểu, không sửa code.

Tinh thần

Một tấm bản đồ, không phải người thay não

Code của mindful-key giống một thành phố: core/ (bộ não C++ dùng chung) làm trung tâm, tỏa ra các khu platforms/ (vỏ macOS/Windows/Android/iOS). Đọc code trực tiếp giống đi bộ dò từng con hẻm — chậm, dễ lạc. Understand-Anything khảo sát cả thành phố 1 lần rồi vẽ bản đồ: click vào 1 tòa nhà (file/hàm/class) là biết nó làm gì, nối với ai, thuộc khu nào.

"Bản đồ giúp đi nhanh hơn — không có nghĩa là khỏi cần biết đường."

Quan hệ với tài liệu tay đã có: OPENKEY-MAP.md, BREATHING-PAUSE-CONTRACT.md... đã viết tay, xác minh theo dòng số cụ thể. Bản đồ AI sinh ra không thay thế mấy file đó — dùng như gương soi: lệch pha thì ghi vào FRICTION-LOG.md, đừng sửa tài liệu gốc theo AI ngay.

Câu lệnh

7 lệnh hay dùng

Gõ thẳng trong Claude Code, không cần cài thêm — plugin đã ở scope project, chỉ áp dụng repo này.

/understand

Quét code, dựng hoặc vá bản đồ. Chạy lần đầu, sau đó để nó tự vá incremental.

/understand-dashboard

Mở lại bản đồ trực quan (web, chạy trên máy) — không quét lại.

/understand-chat "..."

Hỏi nhanh về code, trả lời dựa trên bản đồ đã có sẵn.

/understand-explain <path>

Đào sâu 1 file/hàm cụ thể trước khi sửa — vai trò, láng giềng, luồng dữ liệu.

/understand-diff

Trước khi commit/push: soi thay đổi rung dây chuyền tới đâu.

/understand-domain

Vẽ luồng nghiệp vụ (business flow) — đối chiếu với tài liệu tay.

/understand-onboard

Sinh 1 file Markdown onboarding đầy đủ: layer, tour, file map, điểm phức tạp.

/understand-knowledge · /understand-figma

Không áp dụng — dành cho wiki Karpathy-pattern và file Figma, dự án này không có.

Cơ chế

Lưu ở đâu — có tốn token quét lại không

Đây là phần hay quên nhất: bản đồ không nằm trong bộ nhớ hội thoại — nó nằm trên đĩa, thư mục .ua/ tại gốc repo. Giống viết ra sổ tay để trên bàn thay vì nhớ trong đầu: mở cửa sổ Claude Code mới, sổ vẫn còn nguyên.

1

Chưa đổi commit nào

Nó chỉ hỏi bạn muốn làm gì (rebuild / review / bỏ qua) — không tự đốt token quét lại.

2

Có sửa vài file

Chỉ quét lại đúng file đã đổi (incremental) — rẻ hơn nhiều so với lần đầu.

3

Gõ --full

Bắt buộc quét lại toàn bộ từ đầu — tốn như lần quét đầu tiên. Chỉ dùng khi thật cần.

4

Hỏi/xem bản đồ

understand-chat, understand-dashboard, understand-diff, understand-explain, understand-onboard chỉ đọc file knowledge-graph.json có sẵn — gần như miễn phí, không quét lại.

Bật 1 lần cho khỏe: /understand --auto-update gắn hook chạy sau mỗi commit, tự vá bản đồ mà không cần nhớ tay gọi lại.

Riêng cho mindful-key

Chỗ cần chỉnh tay

Repo này trộn code kỹ thuật (core/, platforms/) chung với web marketing (site/) và build artifact. Quét chung hết sẽ loãng bản đồ, tốn token vô ích. Khi .understandignore hiện ra để duyệt, loại các thư mục:

site/          # web marketing, khác bản chất
build/         # sản phẩm build, không phải nguồn
release-out/   # gói phát hành
models/        # file model/binary

Giữ lại core/, platforms/, docs/, bmad-output/ — vì đó là "vì sao" đằng sau code kỹ thuật.

.gitignore hiện chưa chặn .ua/ — thêm 2 dòng sau để khỏi commit rác tạm, nhưng giữ lại knowledge-graph.json + meta.json để commit thật (đổi máy hay thêm người vào đội thì git pull là có sẵn bản đồ):

.ua/intermediate/
.ua/diff-overlay.json

Hỏi nhanh

Dashboard không mở được?
Kiểm tra .ua/knowledge-graph.json đã tồn tại chưa — nếu chưa, /understand vẫn đang chạy dở hoặc chưa chạy lần nào.
Nghi ngờ mô tả graph sai?
Đọc lại source thật tại filePath ghi trong node đó — mô tả là AI suy diễn, không phải nguồn sự thật. Đúng luật "CẤM đoán" trong CLAUDE.md.
Session mới có phải quét lại không?
Không — bản đồ nằm trên đĩa, không nằm trong bộ nhớ hội thoại. Session mới chỉ đọc file có sẵn.
Nơi lưu artifact?
.ua/ tại gốc repo. Cài scope project, không cộng token always-on vào project khác.
